# 主图设计屏设计规范

## 1. 适用范围与基准

本规范是首页 `#works-wall`（主图设计作品画廊）的视觉与响应式实现基准，覆盖：

- 左侧“主图设计”标题与圆形箭头
- 右侧“电商主图 / MAIN IMAGE DESIGN”标题组
- 三行弧形主图作品墙
- 桌面端、超宽屏、常规笔记本和手机端的比例适配

设计基准画布为 **2560 × 1440 px（16:9）**。文档中的基准像素用于说明设计比例，代码必须优先使用 `vw`、`svh`、`clamp()` 和基于场景宽度的 JavaScript 计算，不得把整屏布局改成固定像素画布。

相关实现：

- `index.html`：标题字符、手写素材和作品卡片内容
- `styles.css`：整屏、安全区、标题与作品墙视觉样式
- `script.js`：卡片尺寸、弧面几何、间距和自动滚动
- `assets/images/main-images/`：主图作品 WebP 素材
- `assets/images/ui/ecommerce-main-image-note.webp`：右侧“电商主图”手写素材

## 2. 画布与安全区

### 2.1 2560 × 1440 基准值

| 项目 | 基准值 | 比例/实现 |
| --- | ---: | --- |
| 屏幕宽高 | 2560 × 1440 px | `100vw × 100svh` |
| 标题左安全边距 | 600 px | 居中 1360 px 标题轨道 |
| 标题右安全边距 | 600 px | 居中 1360 px 标题轨道 |
| 上下设计参考边距 | 200 px | `13.8889svh` |
| 标题网格左边界 | 600 px | 2560 px 基准画布 |
| 标题网格右边界 | 1960 px | 2560 px 基准画布 |
| 标题网格顶部 | 299 px | 高屏使用 `20.7407svh`，低屏回落到 `18.4028svh` |
| 作品墙场景顶部 | 423 px | `29.375svh` |
| 作品墙场景高度 | 1017 px | `70.625svh` |

`.works-gallery-section--main` 必须保持一整屏高度、黑色背景和内容裁切：

```css
width: 100%;
height: 100svh;
min-height: 100svh;
overflow: hidden;
background: #000;
```

桌面端标题轨道与“精选作品”等同类模块共用项目内容宽度 `min(1360px, 53.125vw)`，始终水平居中；在 2560 px 基准画布下左右安全边距均为 600 px。标题、箭头、英文说明和手写素材不得探出该轨道；窄屏按第 7 节规则放宽。

## 3. 左侧“主图设计”标题

### 3.1 结构规则

四个汉字必须保留为四个独立的 `.works-gallery-title__glyph` 元素：

```html
<span class="works-gallery-title__glyph">主</span>
<span class="works-gallery-title__glyph">图</span>
<strong class="works-gallery-title__glyph">设</strong>
<strong class="works-gallery-title__glyph">计</strong>
```

这样可以逐字控制重叠关系，保持设计稿中“贴紧但笔画完整”的排版。禁止重新合并为单个文本节点，也禁止使用整行大幅负 `letter-spacing` 模拟，否则字符会糊叠或重新拉开。

### 3.2 字体与间距

| 属性 | 规范值 |
| --- | --- |
| 字体族 | `var(--font-display)` |
| 字重 | `900` |
| 桌面端字号 | `clamp(48px, min(5.55vw, 10.8svh), 115px)` |
| 行高 | `1` |
| 整行字距 | `0` |
| 相邻汉字负间距 | `-.025em` |
| “主图”颜色 | `#f2f0e8` |
| “设计”颜色 | `#9cff3f` |

中文标题、圆形箭头和右侧英文说明的布局盒底边必须一致；桌面端中文单字使用 `translateY(-.035em)`、英文使用 `translateY(.075em)` 做字体基线的光学校正，使真实字形底部与圆形箭头相差不超过 1 px。

在 2560 × 1440 下，当前校准结果为：

| 测量项 | 结果 |
| --- | ---: |
| 标题组左边界 | 600 px（2560 px 基准画布） |
| 标题行盒高度 | 约 115 px（2560 px 基准画布） |
| 圆形箭头左边界 | 以实机测量为准，保持与“计”之间最多 34 px 布局间距 |
| 含箭头标题组宽度 | 随字体实际渲染变化，不得通过压扁字形强行匹配 |

### 3.3 圆形箭头

| 属性 | 规范值 |
| --- | --- |
| 基准尺寸 | 104 × 104 px |
| 响应式尺寸 | `clamp(42px, min(5vw, 9.6svh), 104px)` |
| 与“计”的距离 | `clamp(18px, 1.45vw, 30px)` |
| 背景色 | `#9cff3f` |
| 箭头色 | `#020302` |
| 圆角 | `50%` |

箭头直径是标题比例的测量锚点。调整汉字间距时不得同步压缩箭头，也不得通过 `scaleX()` 压扁汉字。

## 4. 右侧标题组

右侧标题组必须右对齐到项目内容轨道右边界，即 2560 基准下 x=1960 px。

### 4.1 英文标题

| 属性 | 规范值 |
| --- | --- |
| 内容 | `MAIN IMAGE DESIGN` |
| 字重 | `900` |
| 字号 | `clamp(16px, min(1.85vw, 3.4svh), 40px)` |
| 行高 | `1` |
| 词间距 | `.24em` |
| 字距 | `0` |
| `MAIN IMAGE` | `#f2f0e8` |
| `DESIGN` | `#9cff3f` |

### 4.2 “电商主图”手写素材

- 必须使用 `assets/images/ui/ecommerce-main-image-note.webp`。
- 不允许改回系统行楷字体或重新手写。
- 桌面端宽度：`clamp(104px, 8.9vw, 178px)`。
- 旋转角度：`-5deg`，旋转基点为右下角。
- 素材保持透明背景和原始宽高比，不得拉伸。

## 5. 作品墙视觉规范

### 5.1 场景位置

- 桌面端场景从 `29.375svh` 开始，高度为 `70.625svh`。
- 场景整体使用 `scale(1.1)`，以中心为缩放原点。
- 左右使用黑色渐隐遮罩，中央区域保持清晰。
- 底部使用黑色渐隐，让作品墙自然融入下一屏，而不是出现硬切边。

### 5.2 2560 基准几何

| 参数 | 基准值 |
| --- | ---: |
| 行数 | 3 |
| 每行卡片数 | 17 |
| 总卡片数 | 51 |
| 卡片尺寸 | 280 × 280 px |
| 横向目标间距 | 24 px |
| 纵向间距 | 48 px |
| 圆柱半径 | 1380 px |
| 透视距离 | 1100 px |

非紧凑屏幕的比例公式：

```js
desktopScale = sceneWidth / 2560
cardW = clamp(sceneWidth * 0.109375, 92, 420)
radius = max(520, sceneWidth * 0.5390625)
perspective = max(440, sceneWidth * 0.4296875)
gapX = max(10, 24 * desktopScale)
gapY = max(20, 48 * desktopScale)
```

卡片保持 1:1 正方形。相邻卡片角度必须由卡片宽度、圆柱半径、透视缩放和目标间距动态计算，不允许写死 `angleStep`。

## 6. 色彩与素材

### 6.1 核心颜色

| 用途 | 色值 |
| --- | --- |
| 页面背景 | `#000000` |
| 品牌荧光绿 | `#9CFF3F` |
| 主标题暖白 | `#F2F0E8` |
| 箭头深色 | `#020302` |

品牌绿来自确认后的设计色板，后续不得改成其他近似荧光绿。

### 6.2 图片格式

- 页面实际引用的位图统一使用 WebP。
- 主图素材放在 `assets/images/main-images/`。
- UI 装饰素材放在 `assets/images/ui/`。
- 不得在 HTML 或 CSS 中新增 `.png`、`.jpg`、`.jpeg` 引用。
- 图片不得改变原始宽高比；主图卡片统一以 `object-fit: cover` 或现有卡片规则显示。

## 7. 响应式规则

### 7.1 901 px 及以上

- 保持 2560 基准的比例系统。
- 标题使用居中的 `min(1360px, 53.125vw)` 项目内容轨道，2560 基准下左右边距均为 600 px。
- 标题、箭头、右侧英文和作品墙按视口宽高共同限制，避免在超宽屏或矮屏中失衡。

### 7.2 621–900 px

- 左右安全边距改为 `10vw`。
- 左标题字号上限 76 px。
- 箭头上限 64 px，左间距 12 px。
- 手写素材宽度为 `clamp(86px, 12vw, 110px)`。

### 7.3 620 px 及以下

- 左右边距固定为 24 px，仅作为手机可读性保护。
- 标题顶部改为 `15.5svh`。
- 左标题字号为 `clamp(36px, min(12vw, 9.4svh), 54px)`。
- 右侧标题组移动到左标题下方 32 px，并保持右对齐。
- 作品墙从 `34svh` 开始，高度为 `66svh`，整体缩放为 `1.04`。
- 手写素材宽度为 `clamp(80px, 24vw, 112px)`。

### 7.4 场景宽度小于 700 px

作品墙进入紧凑几何：

```js
cardW = clamp(sceneWidth * 0.3, 108, 128)
radius = 480
perspective = 620
gapX = 12
gapY = 32
```

### 7.5 矮屏保护

当高度不超过 620 px 且宽度大于 620 px：

- 标题顶部调整为 `15svh`。
- 作品墙从 `27svh` 开始，高度为 `73svh`。

## 8. 禁止事项

维护这一屏时禁止：

1. 把 600/600/200 基准边距直接写成所有屏幕固定像素，应保留标题轨道的响应式公式。
2. 合并四个标题字符并用整行大负字距代替逐字间距。
3. 使用 `scaleX()` 压缩汉字来匹配总宽度。
4. 改变箭头直径来掩盖标题间距问题。
5. 用系统字体替换“电商主图”手写 WebP。
6. 在单张卡片上写死 `transform`、`translateZ`、宽高或外边距。
7. 写死弧面 `angleStep`。
8. 新增 PNG/JPG 页面引用。
9. 调整 `styles.css` 或 `script.js` 后忘记递增 `index.html` 中的 `?v=` 版本号。

## 9. 修改后的验收清单

每次修改本屏后至少验证：

- 2560 × 1440
- 1920 × 1080
- 1366 × 768
- 3440 × 1440
- 390 × 844

验收要求：

- 2560 下标题轨道严格位于 x=600–1960 px，左右内容均不越界。
- “主图设计”四字轻微相压但笔画完整，箭头不被挤压。
- 左右标题组不重叠。
- 三行作品卡片之间保留连续黑色间隔。
- 左右边缘作品逐渐变暗，中央作品清晰。
- 页面没有水平滚动条。
- 手机端标题、英文和手写字均完整可读。
- 所有实际引用图片加载成功且格式为 WebP。

## 10. 规范优先级

本文件是 `#works-wall` 的视觉、比例和响应式唯一基准；`docs/gallery-layout-spec.md` 继续用于说明作品墙的弧面计算与防重叠原则。如两份文档中的旧固定数值与当前实现冲突，以本文件和实际比例公式为准。

## 11. 目录区标题复用

首页目录区 `.selected-works-head` 复用本规范的标题系统，但使用目录语义“精选作品”，并保持“左中文标题 / 右英文标题”的横向排版：

- “精选”使用暖白色，“作品”使用品牌荧光绿。
- 四个字保持独立元素，相邻间距为 `-.025em`。
- 字号、行高和圆形箭头尺寸与第 3 节一致。
- 右侧英文为 `SELECTED WORKS`，其中 `SELECTED` 使用暖白色，`WORKS` 使用品牌荧光绿，字体、粗细、无负字距逻辑对齐主图页 `MAIN IMAGE DESIGN`。
- 桌面端标题栏与下方目录模块保留约 `108px–164px` 的响应式距离，避免标题压住作品卡片。
- 900px 以下改为上下堆叠，英文左对齐，标题与卡片距离收窄但不贴边。
- 目录标题样式由 `card-radius-system.css` 维护；修改后递增该文件在 `index.html` 中的版本号。
