# 3D 作品墙布局规范

> 当前 `#works-wall` 的整屏视觉、标题排版和响应式数值，以 [`main-image-screen-design-spec.md`](./main-image-screen-design-spec.md) 为准。本文件主要保留弧面计算、防重叠和内容维护原则；旧的固定桌面参数仅作为 2560 × 1440 基准参考。

## 适用范围

本规范适用于首页中的 `#works-wall` 三行弧形作品墙。相关实现位于：

- `index.html`：作品卡片原始内容
- `styles.css`：场景、卡片尺寸及视觉样式
- `script.js`：卡片复制、弧形排列、间距和自动滚动

## 固定布局参数

桌面端基准参数：

| 参数 | 数值 | 说明 |
| --- | ---: | --- |
| 行数 | 3 | 不随屏幕变化 |
| 每行卡片数 | 17 | 包含自动复制的卡片 |
| 卡片最大尺寸 | 280 × 280 px | 卡片始终为正方形 |
| 可见横向间距 | 24 px | 相邻卡片投影后的目标空隙 |
| 纵向间距 | 48 px | 相邻行之间的本地坐标空隙 |
| 圆柱半径 | 1380 px | 控制弧面曲率 |
| 透视距离 | 1100 px | 必须与 `.works-gallery-scene` 一致 |

响应式参数：

| 视口/场景宽度 | 横向间距 | 纵向间距 | 圆柱半径 |
| --- | ---: | ---: | ---: |
| 小于 760 px | 12 px | 34 px | 900 px |
| 760–1099 px | 18 px | 42 px | 1160 px |
| 1100 px 及以上 | 24 px | 48 px | 1380 px |

## 防重叠规则

1. 不允许手动写死 `angleStep`。
2. 横向角度必须根据卡片宽度、圆柱半径、透视缩放和目标间距动态计算。
3. 卡片宽度变化后必须重新运行 `updateGalleryLayout()`。
4. hover 状态不得使用 `translateZ()` 将整张卡片向前移动。
5. hover 只能调整亮度、阴影或卡片内部画面的缩放。
6. 三行卡片的垂直中心距离必须为 `cardH + gapY`。
7. 鼠标控制的整体旋转上限为 X 轴 5°、Y 轴 18°，避免透视压缩造成视觉相交。

## 横向角度计算

当前代码先计算圆柱中心位置的透视缩放：

```js
centerScale = perspective / (perspective + radius)
```

再把屏幕目标间距换算为圆柱本地间距：

```js
localGapX = gapX / centerScale
```

最终使用卡片宽度和本地间距计算相邻卡片角度：

```js
angleStep =
  asin((cardW + localGapX) / radius) * 180 / PI
```

这样在卡片尺寸随视口变化时，卡片之间仍会保留稳定空隙。

## 修改检查清单

修改作品墙后至少检查以下尺寸：

- 1920 × 1080
- 1440 × 900
- 1024 × 768
- 390 × 844

每个尺寸都应满足：

- 同一行相邻卡片没有覆盖
- 三行之间存在连续黑色间隔
- 自动滚动时没有突然跳动
- hover 时不覆盖左右卡片
- 页面没有横向滚动条
- 首尾循环看不出明显断点

## 内容维护

新增作品时，只需要在 `index.html` 的 `[data-gallery-wrapper]` 内增加一个
`[data-gallery-card]`。脚本会自动复制内容补足 51 张卡片，不需要手动复制三行。

不要在单张卡片上设置：

- 固定 `transform`
- 自定义 `translateZ`
- 自定义宽高
- 负 margin

这些属性会破坏统一间距和弧形排列。
