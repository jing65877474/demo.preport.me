# EC DESIGN 拼贴屏规范

## 适用范围

本规范适用于首页 `#creative-collage` 拼贴屏。

相关文件：

- `index.html`：10 张卡片与中心标题结构
- `collage-arc.css`：舞台高度、卡片尺寸、标题和背景扫光
- `script.js`：轨道坐标、入场、视差、悬停、拖拽与惯性

## 布局基准

拼贴屏使用 14 等分椭圆轨道，仅占用上下两段各 5 个槽位：

```js
slotIds = [-2, -1, 0, 1, 2, 5, 6, 7, 8, 9]
slotAngle = Math.PI * 2 / 14
```

卡片必须保持正方形，尺寸由 CSS 变量统一控制：

```css
--arc-card-size: clamp(118px, 10.2vw, 176px);
```

禁止在单张 `.collage-card` 上单独设置宽度、高度、外边距或固定定位坐标。

## 轨道间距

横向间距由 `script.js` 的 `orbitSize()` 统一控制。

| 舞台宽度 | 横向半径 | 最大值 | 纵向半径 |
| --- | ---: | ---: | ---: |
| `<= 560px` | 宽度的 `44%` | `176px` | 高度的 `34%`，最大 `278px` |
| `561-900px` | 宽度的 `39%` | `325px` | 高度的 `35%`，最大 `330px` |
| `> 900px` | 宽度的 `36%` | `600px` | 高度的 `35%`，最大 `380px` |

调整左右间距时，只修改横向半径 `x`。不要通过缩小卡片、修改槽位数量或压缩舞台高度实现。

建议每次调整不超过 3 个百分点，避免卡片与中心标题相交。

## 舞台尺寸

桌面端舞台高度：

```css
height: clamp(980px, 125svh, 1120px);
min-height: 980px;
```

平板端最小高度为 `820px`，手机端固定高度为 `820px`。

舞台必须保留 `overflow: visible`，避免旋转后的卡片边角被裁切。

## 动效规范

必须保留以下效果：

1. 卡片从中心螺旋展开到上下弧线。
2. 中心标题同步缩放、去模糊并渐显。
3. 鼠标移动产生轻量视差。
4. 悬停卡片突出，邻近卡片柔和让位。
5. 拖拽改变轨道角度。
6. 松手后保留衰减惯性。
7. 离开后返回拼贴屏时重新播放入场。
8. 背景扫光保持在卡片下方，不得遮挡卡片。

不要加入无人操作时的自动旋转。

## 层级规则

- 背景与扫光：`z-index: 0`
- 卡片：由脚本按深度动态计算
- 中心标题：`z-index: 30`
- 页面固定导航：保持项目现有最高层级

悬停时允许提高当前卡片层级，但不能通过覆盖相邻卡片制造突出效果。

## 响应式检查

每次修改至少检查：

- 1920 × 1080
- 1440 × 900
- 1024 × 768
- 390 × 844

检查要求：

- 10 张卡片全部可见
- 左右间距紧凑但不重叠
- 卡片不与中心标题相交
- 卡片边角不被舞台裁切
- 入场、悬停、视差、拖拽、惯性均正常
- 页面没有横向滚动条
- 控制台无错误
